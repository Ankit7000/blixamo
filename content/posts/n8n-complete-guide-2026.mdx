## What Is n8n and Why It Wins in 2026

n8n (pronounced "n-eight-n") is a source-available workflow automation tool that you can self-host. Think of it as Zapier but running on your own server — you control the data, the rate limits, and the cost.

The key difference from Zapier in 2026: n8n has a code node. Every workflow can drop into JavaScript or Python at any point. Zapier is drag-and-drop only. When you need custom logic — parse a weird API response, do string manipulation, call a custom function — n8n handles it. Zapier doesn't.

**n8n usage stats as of early 2026:**
- 400+ integrations (Zapier has 6,000+, but n8n covers 95% of common developer use cases)
- 45,000+ GitHub stars
- Used by teams at Siemens, Thomson Reuters, and thousands of startups
- 300+ new community workflow templates added monthly

The self-hosted version is free under n8n's Sustainable Use License. This means you can use it for any internal automation. You cannot build a product on top of n8n and sell it without a commercial license — which is fine for indie developers.

I compared all three automation tools in detail in [n8n vs Make vs Zapier for indie developers](/blog/n8n-vs-make-vs-zapier-indie-dev). If you want the full breakdown before committing, read that first. Short answer: for developers who can self-host, n8n wins.

## Step 1 of 5 — Deploy n8n with Docker Compose

Create the n8n directory and compose file:

```bash
mkdir -p /opt/n8n && cd /opt/n8n
```
Create the Docker Compose file:
#
```bash
# /opt/n8n/docker-compose.yml
version: "3.8"
# ---
services:
  n8n:
    image: n8nio/n8n:latest
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=n8n.yourdomain.com
      - N8N_PORT=5678
      - N8N_PROTOCOL=https
      - NODE_ENV=production
      - WEBHOOK_URL=https://n8n.yourdomain.com/
      - GENERIC_TIMEZONE=Asia/Kolkata
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=your-strong-password
    volumes:
      - n8n_data:/home/node/.n8n
# ---
volumes:
  n8n_data:
    driver: local
```
```bash
# Start n8n
docker compose up -d
```
<Callout type="tip" title="Expected output">
[+] Running 2/2
 ✔ Network n8n_default  Created
 ✔ Container n8n-n8n-1  Started
</Callout>
#
**Why this matters:** The `WEBHOOK_URL` setting is critical. n8n generates webhook URLs for every trigger node. If this isn't set to your public domain, webhook callbacks from external services (WhatsApp, GitHub, Stripe) will fail silently — one of the most common n8n setup mistakes.

## Step 2 of 5 — Configure SSL with Nginx
#
n8n runs on port 5678. You need nginx to proxy it over HTTPS. If you're using Coolify, you can add n8n as a resource and Coolify handles SSL automatically. For manual setup:
#
```bash
# Install nginx and certbot
apt install nginx certbot python3-certbot-nginx -y
```
Create the nginx config:
#
```bash
# /etc/nginx/sites-available/n8n
server {
    server_name n8n.yourdomain.com;
# ---
    location / {
        proxy_pass http://localhost:5678;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
        # Required for n8n websockets
        proxy_read_timeout 86400;
    }
# ---
    listen 80;
}
```
```bash
# Enable config and get SSL cert
ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
certbot --nginx -d n8n.yourdomain.com --non-interactive \
  --agree-tos -m your@email.com
```
<Callout type="tip" title="Expected output">
Successfully received certificate.
Certificate is saved at: /etc/letsencrypt/live/n8n.yourdomain.com/fullchain.pem
</Callout>
#
Open `https://n8n.yourdomain.com` — you should see the n8n login screen. Enter the basic auth credentials you set in the compose file.

## Step 3 of 5 — Build Your First Workflow
#
n8n workflows are visual chains of nodes. Each node does one thing: trigger, transform, or act.
#
**Workflow 1: Webhook to Telegram notification**
#
This is the simplest useful workflow — receive any HTTP POST and forward it as a Telegram message. I use this for deployment notifications from Coolify.
#
In n8n:
1. Add node: **Webhook** (trigger) — set path to `/deploy-notify`
2. Add node: **Telegram** → action: Send Message → chat ID: your chat ID → message: `{{ $json.body.status }} — {{ $json.body.app }}`
3. Click **Save** and **Activate**
#
Your webhook URL is: `https://n8n.yourdomain.com/webhook/deploy-notify`
#
```bash
# Test it with curl
curl -X POST https://n8n.yourdomain.com/webhook/deploy-notify \
  -H "Content-Type: application/json" \
  -d '{"status": "deployed", "app": "blixamo"}'
```
<Callout type="tip" title="Expected output">
Telegram message received: "deployed — blixamo"
</Callout>
#
**Workflow 2: Scheduled database backup to email**
#
This workflow runs every day at 2am, dumps your PostgreSQL database, and emails you a summary.
#
1. Node: **Schedule Trigger** — set to `0 2 * * *` (2am daily)
2. Node: **Execute Command** — command: `pg_dump -U postgres mydb | gzip > /tmp/backup_$(date +%Y%m%d).sql.gz && echo "Backup done: $(ls -lh /tmp/backup_*.sql.gz | tail -1)"`
3. Node: **Gmail** → send backup summary to your email
#
I use a version of this for blixamo's content backup. All on the same VPS, no extra cost.
#
**Workflow 3: WhatsApp AI assistant via Claude API**
#
This is my most-used workflow. A WhatsApp message comes in via webhook → n8n sends it to Claude API → response goes back to WhatsApp. Under 3 seconds round-trip.
#
I covered this in full detail in the [WhatsApp AI assistant with n8n and Claude API](/blog/whatsapp-ai-assistant-n8n-claude-api) guide. The relevant n8n nodes are:
#
1. **Webhook** trigger (WhatsApp sends POST to your n8n URL)
2. **HTTP Request** node → Claude API (`claude-sonnet-4-5`)
3. **HTTP Request** node → WhatsApp Business API (send reply)
#
The code node in step 2 handles extracting the message text and formatting the API request — something Zapier can't do without a paid Zap code step.

## Step 4 of 5 — Connect Your Apps
#
n8n has 400+ built-in integrations. Here are the ones I use most and how to connect them.
#
**Claude API (Anthropic):**
#
HTTP Request node → URL: `https://api.anthropic.com/v1/messages` → Headers: `x-api-key: YOUR_KEY`, `anthropic-version: 2023-06-01` → Body: JSON with model and messages array.
#
Store your API key in n8n Credentials (not hardcoded in the workflow).
#
**Gmail:**
#
n8n → Credentials → Add → Gmail OAuth2. Follow the Google OAuth flow. Takes 5 minutes.
#
**PostgreSQL:**
#
n8n → Credentials → Add → Postgres → enter connection string. n8n can query, insert, and update rows directly in workflows.
#
**GitHub Webhooks:**
#
In GitHub repo → Settings → Webhooks → add your n8n webhook URL. Select events (push, PR, release). n8n triggers on every event.
#
**Airtable / Notion / Supabase:**
#
All have native n8n nodes. Add API key in Credentials. Supabase works via the Postgres node using the Supabase connection string.

## Step 5 of 5 — Production Hardening
#
Before relying on n8n for critical automations, add these:
#
```bash
# 1. Confirm n8n restarts automatically after VPS reboot
docker compose ps
# Should show "Up" status — restart: always in compose handles this
# ---
# 2. Check n8n data volume is persisted
docker volume inspect n8n_n8n_data
# Should show Mountpoint: /var/lib/docker/volumes/n8n_n8n_data/_data
# ---
# 3. Monitor n8n memory usage
docker stats n8n-n8n-1 --no-stream
```
<Callout type="tip" title="Expected output">
CONTAINER   CPU %   MEM USAGE / LIMIT   MEM %
n8n-n8n-1   0.1%    82MiB / 3.8GiB      2.1%
</Callout>
#
<Callout type="warning" title="n8n memory during complex workflows">
n8n idles at ~80MB but can peak to 250-350MB when processing large payloads or running many parallel workflows. On a 4GB VPS this is fine. On a 1GB VPS (AWS Lightsail $5), n8n will frequently OOM. The 4GB Hetzner CPX22 at the same price handles this comfortably.
</Callout>
#
**Backup your workflows:**
#
n8n exports all workflows as JSON. Do this monthly:
#
```bash
docker exec n8n-n8n-1 n8n export:workflow \
  --all --output=/home/node/.n8n/workflows-backup.json
# Copy backup off-server
scp root@YOUR_VPS:/var/lib/docker/volumes/n8n_n8n_data/_data/workflows-backup.json ./
```

## Troubleshooting n8n
#
**Problem: Webhooks not receiving data**
#
```bash
# Check n8n is actually listening on 5678
netstat -tlnp | grep 5678
# Check nginx is proxying correctly
curl -I https://n8n.yourdomain.com
```
Most common cause: `WEBHOOK_URL` in docker-compose.yml doesn't match your actual domain. Rebuild after fixing:
#
```bash
docker compose down && docker compose up -d
```
**Problem: n8n keeps disconnecting from workflow editor**
#
This is a WebSocket issue. Make sure your nginx config has `proxy_read_timeout 86400` and the Upgrade/Connection headers set. The config in Step 2 of this guide includes all required headers.
#
**Problem: Workflow runs but no output**
#
Enable **Execution logging** in n8n Settings → Log Level → Debug. Check the execution log for the specific node that's failing. 90% of the time it's a credential issue or a missing field in the JSON path.
#
**Problem: n8n container restarts every few hours**
#
```bash
docker logs n8n-n8n-1 --tail 50
```
Usually OOM. Check `docker stats` during a workflow run. If RAM spikes to container limit, increase Docker memory limit or add more swap to the VPS.

## Decision Framework
#
**Use self-hosted n8n if:**
- You're a developer comfortable with a VPS and Docker
- You have more than 2-3 automations (cost savings compound fast)
- You need custom JavaScript/Python logic in workflows
- Data privacy matters — you control where everything is stored
#
**Use Zapier if:**
- You need 1-2 simple automations and don't want to manage a server
- You need integrations from Zapier's 6,000+ app library that n8n doesn't have
- You're non-technical and need a zero-config solution
#
**Use Make if:**
- You want a middle ground — no self-hosting but more power than Zapier
- You're comfortable with a visual interface and limited code

## FAQ
#
### Is n8n really free to self-host?
#
Yes — the community edition is free under n8n's Sustainable Use License. You can run unlimited workflows, unlimited executions, and connect unlimited apps. The paid cloud version (n8n Cloud) starts at €20/month. Self-hosting costs only your VPS — Rs 465/month on Hetzner.
#
### How reliable is self-hosted n8n?
#
My n8n instance has been running for 6 months without unexpected downtime. Docker's `restart: always` handles crashes. The main risk is your VPS going down — mitigated by Hetzner's 99.9% SLA and PM2/systemd restart policies.
#
### Can n8n handle high volume automations?
#
Yes, with caveats. A single n8n instance on CPX22 can comfortably handle 10,000+ executions/day. For higher volume, n8n supports queue mode with Redis (which you're likely already running). I cover the Redis queue setup in the [self-hosting n8n guide](/blog/self-hosting-n8n-hetzner-vps).
#
### Does n8n work with the Claude API?
#
Yes — n8n's HTTP Request node connects to any REST API including Anthropic's Claude API. I use it for my WhatsApp AI assistant workflow. Full integration guide in [WhatsApp AI assistant with n8n and Claude API](/blog/whatsapp-ai-assistant-n8n-claude-api).
#
### What happens to my data when n8n runs?
#
With self-hosted n8n, all execution data stays on your VPS. n8n stores workflow execution logs in its SQLite or Postgres database (your choice). Nothing leaves your server unless your workflow explicitly sends data to an external service.
#
### Can I import Zapier Zaps into n8n?
#
No direct import tool exists. You'll need to recreate workflows manually in n8n. For simple 2-3 step Zaps, this takes 10-15 minutes each. For complex ones, budget 30-60 minutes. The migration pays back in Rs 7,400/month savings from week one.
