## Why Telegram Instead of a Web App

A Telegram bot installs with one click — users tap a link, tap Start, and they're using your product. No signup form, no email verification, no password to remember. For personal productivity tools and niche AI assistants, this frictionless onboarding beats a full web app.

The infrastructure advantage matters too. This bot runs as a single Python process consuming ~50MB RAM on the same Hetzner CPX22 that already runs blixamo.com, n8n, and three other projects. Total additional cost: Rs 0 in hosting, Rs 600–725/month in Claude API tokens at 200 messages/day.

I use mine daily: send any article URL and get a 5-bullet summary in 10 seconds, ask for quick API cost estimates, draft tweet threads for new posts. Tasks I'd otherwise switch apps for — the bot keeps them in one place.

## Step 3 of 6 — The Complete Bot Code
#
Create `/home/ankit/telegram-bot/bot.py` and paste this entire file. It's 114 lines — all config, guards, the Claude call, and handlers in one place.
#
<Callout type="info" title="Four Things Worth Noting">
- `allowed()` checks `ALLOWED_USER_ID` from `.env`. Set to your Telegram numeric ID for a private bot. Set to `0` to allow anyone.
- `rate_limited()` enforces a 3-second gap between requests per user — prevents API cost overruns.
- `trim()` keeps conversation history at 2× `MAX_HISTORY` entries to control token costs.
- `ask_claude()` is synchronous by design — aiogram runs it in a thread pool automatically so the event loop stays unblocked.
</Callout>
#
```bash
import asyncio, os, logging, time
from collections import defaultdict
from dotenv import load_dotenv
import anthropic
from aiogram import Bot, Dispatcher, F
from aiogram.filters import CommandStart, Command
from aiogram.types import Message
#
load_dotenv()
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)
#
# ── Config ──────────────────────────────────────────────
BOT_TOKEN        = os.getenv("BOT_TOKEN")
ANTHROPIC_KEY    = os.getenv("ANTHROPIC_API_KEY")
ALLOWED_USER_ID  = int(os.getenv("ALLOWED_USER_ID", "0"))
CLAUDE_MODEL = os.getenv(
    "CLAUDE_MODEL",
    "claude-haiku-3-5-20251001"
)
MAX_HISTORY = int(os.getenv("MAX_HISTORY_MESSAGES", "10"))
RATE_LIMIT_SECS  = 3
#
bot    = Bot(token=BOT_TOKEN)
dp     = Dispatcher()
claude = anthropic.Anthropic(api_key=ANTHROPIC_KEY)
history      : dict[int, list]  = defaultdict(list)
last_request : dict[int, float] = defaultdict(float)
#
SYSTEM_PROMPT = (
    "You are a helpful AI assistant in a Telegram bot. "
    "Keep replies concise — under 300 words "
    "unless asked for more. "
    "Use plain text. Use code blocks only for code."
)
#
# ── Guards ──────────────────────────────────────────────
def allowed(user_id: int) -> bool:
    allowed = ALLOWED_USER_ID
    return allowed == 0 or user_id == allowed
#
def rate_limited(user_id: int) -> bool:
    now = time.time()
    if now - last_request[user_id] < RATE_LIMIT_SECS:
        return True
    last_request[user_id] = now
    return False
#
def trim(user_id: int):
    if len(history[user_id]) > MAX_HISTORY * 2:
        cap = MAX_HISTORY * 2
    history[user_id] = history[user_id][-cap:]
#
# ── Claude call ─────────────────────────────────────────
def ask_claude(user_id: int, user_text: str) -> str:
    history[user_id].append({
        "role": "user",
        "content": user_text
    })
    trim(user_id)
    try:
        resp = claude.messages.create(
            model=CLAUDE_MODEL,
            max_tokens=1024,
            system=SYSTEM_PROMPT,
            messages=history[user_id],
        )
        reply = resp.content[0].text
    except anthropic.RateLimitError:
        return "⚠️ Rate limit hit — try again in a moment."
    except anthropic.APIError as e:
        logger.error("API error: %s", e)
        return "❌ API error. Check your key and credits."
    except Exception as e:
        logger.error("Unexpected: %s", e)
        return "❌ Something went wrong. Try again."
    history[user_id].append({
        "role": "assistant",
        "content": reply
    })
    return reply
#
# ── Handlers ────────────────────────────────────────────
@dp.message(CommandStart())
async def cmd_start(m: Message):
    if not allowed(m.from_user.id): return
    await m.answer(
        "👋 Powered by Claude.\n\n"
        "/help — commands\n"
        "/clear — wipe history\n"
        "/summarise [text] — bullet summary\n\n"
        "Or just send any message."
    )
#
@dp.message(Command("help"))
async def cmd_help(m: Message):
    if not allowed(m.from_user.id): return
    await m.answer(
        "/start  /clear  /summarise [text]\n"
        "Or chat freely."
    )
#
@dp.message(Command("clear"))
async def cmd_clear(m: Message):
    if not allowed(m.from_user.id): return
    history[m.from_user.id].clear()
    await m.answer("✅ History cleared.")
#
@dp.message(Command("summarise"))
async def cmd_summarise(m: Message):
    if not allowed(m.from_user.id): return
    text = m.text.replace("/summarise", "").strip()
    if not text:
        await m.answer(
            "Usage: /summarise [paste text here]"
        )
        return
    await m.answer("⏳ Summarising...")
    prompt = f"Summarise in 5 bullet points:\n\n{text}"
    reply = ask_claude(m.from_user.id, prompt)
    await m.answer(reply)
#
@dp.message(F.text)
async def handle_message(m: Message):
    if not allowed(m.from_user.id): return
    if rate_limited(m.from_user.id):
        await m.answer("⏳ Too fast — wait a moment.")
        return
    await bot.send_chat_action(m.chat.id, "typing")
    reply = ask_claude(m.from_user.id, m.text)
    chunks = [
        reply[i:i+4000]
        for i in range(0, len(reply), 4000)
    ]
    for chunk in chunks:
        await m.answer(chunk)
#
# ── Entry point ─────────────────────────────────────────
async def main():
    logger.info("Bot starting...")
    await dp.start_polling(bot)
#
if __name__ == "__main__":
    asyncio.run(main())
```

## Step 5 of 6 — Deploy as a systemd Service
#
Running the bot in a terminal session means it stops when you disconnect. systemd keeps it running permanently and restarts it automatically on crashes — which is why we use it instead of a simple `nohup` command.
#
```bash
sudo cat > /etc/systemd/system/telegram-bot.service << 'EOF'
[Unit]
Description=Telegram Claude Bot
After=network.target
#
[Service]
Type=simple
User=ankit
WorkingDirectory=/home/ankit/telegram-bot
ExecStart=/home/ankit/telegram-bot/venv/bin/python bot.py
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
EnvironmentFile=/home/ankit/telegram-bot/.env
#
[Install]
WantedBy=multi-user.target
EOF
#
sudo systemctl daemon-reload
sudo systemctl enable telegram-bot
sudo systemctl start telegram-bot
```
<Callout type="tip" title="Expected output">
#
Created symlink ... → .../systemd/system/...
#
</Callout>
Check it's running:
#
```bash
sudo systemctl status telegram-bot
sudo journalctl -u telegram-bot -f
```
<Callout type="tip" title="Expected output">
Status shows `active (running)`. Logs show `Bot starting...`. If status is `failed`, the most common cause is wrong Python path — verify `ExecStart` points to your venv Python.
</Callout>

You should see `Bot starting...` in the logs. Memory usage: ~50MB Python process. On a Hetzner CPX22 running other services at ~1.5GB total, this is negligible.

async def fetch_article_text(url: str) -> str:
    async with aiohttp.ClientSession() as session:
        async with session.get(url, timeout=10) as resp:
            html = await resp.text()
    soup = BeautifulSoup(html, 'html.parser')

## Troubleshooting Common Issues
#
#
![Telegram bot architecture — aiogram → Python handler → Claude Haiku API → reply in under 3s](/images/posts/build-telegram-bot-claude-api-python/03-bot-architecture.png)
*Bot architecture — build telegram bot claude api python*
#
**Bot starts but doesn't respond to messages**
Most likely cause: wrong `BOT_TOKEN` in `.env`. Verify the token matches exactly what BotFather gave you. Tokens expire if you regenerate them in BotFather — the old token stops working immediately.
#
**`ModuleNotFoundError: No module named 'aiogram'`**
The systemd service isn't using the venv Python. Verify `ExecStart` points to `/home/ankit/telegram-bot/venv/bin/python`, not `/usr/bin/python3`.
#
**Claude returns empty responses**
Check your Anthropic API key has credits. Log into console.anthropic.com and verify the balance. New accounts require a $5 minimum top-up — there's no free tier.
#
**Bot crashes after a few hours**
Check `journalctl -u telegram-bot --since "1 hour ago"` for the error. Common causes: memory leak from unbounded history growth (the `trim()` function handles this — verify it's being called), or a network timeout that aiogram doesn't recover from (the `Restart=always` in systemd handles this).
#
**`ConflictError: 409 Conflict`**
Two instances of the bot are running simultaneously — your local test and the deployed service. Run:
#
```bash
sudo systemctl stop telegram-bot
# Kill any local instances
sudo systemctl start telegram-bot
```

### How much does a personal Telegram bot with Claude Haiku cost per month?
#
At 200 messages per day with average 300 output tokens per response: 200 × 300 × 30 = 1,800,000 output tokens. Claude Haiku 3.5 costs $4/million output tokens = $7.20/month (~Rs 605). Including input tokens: total ~$8.64/month (~Rs 725). The [full API cost breakdown](/blog/claude-api-vs-openai-cost-india) covers all model options.
#
### Can I run this on a free server instead of Hetzner?
#
Oracle Cloud Free Tier gives an ARM64 instance with 1GB RAM that runs this bot comfortably. Railway.app free tier works for testing but sleeps after inactivity — the bot stops responding until it wakes up. For always-on operation, a VPS is the right choice. If you already have a Hetzner server, adding this bot costs nothing extra.
#
### How do I share the bot with friends without removing restrictions?
#
Change `ALLOWED_USER_ID=0` to disable the restriction entirely, or maintain a comma-separated list: `ALLOWED_USER_IDS=123456789,987654321` and update `allowed()` to check against the list.
#
### Does conversation history persist across bot restarts?
#
No. The current implementation stores history in Python dictionaries (in-memory). When the bot restarts, history clears. For persistent history, replace the `defaultdict` with a SQLite database using `aiosqlite`. Each message insert takes ~1ms — negligible performance impact.
#
### Can I add the bot to a group chat?
#
Yes. Add the bot to a Telegram group and it will respond to all messages. For group use, store history by `chat.id` rather than `user.id` for shared group context, or by `user.id` for private context within the group.
#
### How is this different from the WhatsApp AI assistant guide?
#
The [WhatsApp AI assistant](/blog/whatsapp-ai-assistant-n8n-claude-api) uses n8n (no-code), WAHA (WhatsApp HTTP API), and Claude — more complex setup, no coding required. This Telegram bot uses Python directly — simpler infrastructure, requires writing code, but more customisable with better free tooling.
