## Why Most SaaS Stacks Are Overengineered for MVPs

Before the stack, let me explain why the default advice is wrong.

Everyone tells you to use AWS, Vercel, PlanetScale, and Render. These are all excellent services. They are also priced for companies that have revenue, not indie developers testing ideas.

You don't need autoscaling at 0 users. You don't need 99.999% uptime SLAs when you have no paying customers yet. You need to ship fast, keep costs near zero, and find out if anyone actually wants the thing.

The zero-budget stack is optimized for exactly that: **ship fast, pay almost nothing, scale when you have customers**.

### Coolify — Zero-Downtime Deploys Without DevOps

**Cost:** Free (self-hosted)

Coolify is the tool that makes this stack practical. It is an open-source Heroku/Vercel alternative that you run on your own VPS. Connect your GitHub repo, push code, Coolify handles the build + deploy + SSL automatically.

Before Coolify, deploying meant SSHing into the server, running `git pull`, restarting PM2, and hoping nothing broke. Now it is a GitHub push. The [Coolify + Hetzner deployment guide](/blog/deploy-nextjs-coolify-hetzner) has the full 25-minute setup.

### Next.js 15 — Frontend + API in One

**Cost:** Free

Next.js App Router handles both the frontend and backend API routes in one codebase. No separate Express server, no separate API deployment. One repo, one deploy, one process.

For SaaS MVPs specifically, Next.js is ideal because:
- API routes handle auth, payments, data mutations
- Server components fetch data without a separate API call
- Built-in image optimization
- ISR (Incremental Static Regeneration) for near-static performance

### Supabase Free Tier — Database + Auth + Storage

**Cost:** Free up to 500MB database, 1GB storage, 50K monthly active users

Supabase gives you a managed Postgres database, authentication (email, Google, GitHub OAuth), file storage, and realtime subscriptions — all free up to generous limits.

For a new SaaS MVP, you will not hit the limits for months. When you do, the Pro plan is $25/month — only pay when you have users who justify it.

<Callout type="warning" title="Supabase free tier pauses after 1 week of inactivity">
Supabase free tier databases pause if there is no activity for 7 days. Set up a simple cron job to ping the database once a day, or upgrade to Pro ($25/month) when you have real users.
</Callout>

### n8n — Automation Without Zapier's $300/Month Bill

**Cost:** Free (self-hosted on the same Hetzner VPS)

n8n replaces Zapier, Make, and most custom webhook scripts. Welcome emails, payment confirmations, Slack notifications, data sync — all of it runs in n8n on the same VPS.

I replaced $240/year in Zapier costs with a Docker container that uses 200MB of RAM. The [self-hosting n8n on Hetzner guide](/blog/self-hosting-n8n-hetzner-vps) has the complete Docker setup.

### Cloudflare Free — CDN, DDoS Protection, DNS

**Cost:** Free

Every SaaS needs a CDN and DDoS protection. Cloudflare's free tier gives you both, plus: SSL certificates, caching rules, Web Analytics (free, no cookies, GDPR-compliant), and bot protection.

Point your domain to Cloudflare, set proxied DNS records, and your VPS is protected behind Cloudflare's network. Anyone trying to find your server IP to DDoS it can't — they only see Cloudflare.

### Resend — Transactional Email That Actually Delivers

**Cost:** Free up to 3,000 emails/month

Resend is the cleanest transactional email API I have used. One API call, email delivered to inbox (not spam). The free tier handles 3,000 emails/month — enough for 1,000+ active users receiving weekly emails.

## Troubleshooting Common Setup Issues
#
### Coolify SSL certificate fails
#
Make sure your domain DNS is pointing to the VPS IP and not behind Cloudflare proxy (orange cloud) when first getting the SSL cert. Switch Cloudflare to DNS-only (grey cloud), get the cert, then re-enable proxy.
#
### Supabase connection pooling errors
#
By default Next.js serverless functions create a new DB connection per request. Use `@supabase/ssr` with connection pooling enabled in Supabase dashboard under Settings → Database → Connection Pooling.
#
### n8n workflows not triggering
#
n8n webhooks require a publicly accessible URL. Make sure Coolify has set up the reverse proxy for n8n correctly — the URL `https://n8n.yourdomain.com/webhook/...` must be reachable from the internet.
