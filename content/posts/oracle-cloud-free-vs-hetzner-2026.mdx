**The free ARM instance (what I use):**

## ARM64 Compatibility — The Real Oracle Free Gotcha

Oracle's free instances are ARM64 architecture. Most common packages work. But you will hit ARM64 incompatibilities with some Docker images and Node.js native modules.

**What works without issues on ARM64:**
- Python 3.x (pip install everything standard)
- Node.js 18/20 LTS
- Official Docker images with multi-arch support (n8n, Redis, Nginx)
- Go binaries (most compile for ARM64)

**What breaks or needs workarounds on ARM64:**
- Some older npm packages with native bindings
- Certain Docker images without ARM64 variants
- Chrome/Puppeteer (needs special ARM builds)
- Some database drivers

<Callout type="warning" title="Always Check ARM64 Support Before Committing">
Before deploying a stack to Oracle Free ARM, verify every Docker image has an ARM64 variant by checking Docker Hub for `linux/arm64` in the supported platforms. An image without ARM64 support will silently fail with `exec format error` — not immediately obvious if you're used to x86.
</Callout>

Hetzner CPX22 runs AMD EPYC (x86_64). Every package, Docker image, and binary works without compatibility checks. This is the silent time cost of Oracle Free: debugging ARM64 issues that wouldn't exist on x86.

## Troubleshooting Common Issues

**Oracle Free: Account stuck in pending after card verification**

Oracle's fraud detection blocks many international cards. Try a Wise virtual Visa card — it has the highest success rate for Oracle's verification process. If Wise fails, try a Niyo Global card. Standard debit cards from many countries (including SBI, HDFC) often fail the verification.

**Oracle Free: Instance shows "Out of host capacity"**

Oracle's free tier regions have limited capacity. Common fix: try a different availability domain within the same region. Mumbai region (ap-mumbai-1) often has availability issues — try Hyderabad (ap-hyderabad-1) or an international region.

**Oracle Free: Node.js/Python package won't install on ARM64**

Check if the package has ARM64 wheels available. For Python: `pip install package` then check if it downloaded a `cp3x-manylinux_aarch64` wheel. For Node.js: if `npm install` fails on native modules, check the package's GitHub for ARM64 support issues.

**Hetzner: First server creation blocked**

New Hetzner accounts without payment history may require a small prepaid balance before creating servers. Go to Account → Payment → Add Balance → add €5. This confirms your payment method and unlocks server creation.
