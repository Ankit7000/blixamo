## Why Plausible Over GA4 in 2026

GA4's problems are well-documented by now. Sessions replaced pageviews as the primary metric. The interface requires 6 clicks to find what GA3 showed on the home screen. Every deployment needs a cookie consent banner under GDPR — which tanks conversion rates by 15-25% according to multiple A/B tests.

Plausible solves all of this by not using cookies at all. It fingerprints sessions via a hash of IP + user agent + date — no persistent identifiers stored. This means no cookie banner needed in the EU under current GDPR guidance (though consult your legal team for compliance decisions).

The Plausible script is 1KB. GA4's gtag.js is 45KB. On a 3G connection, that's the difference between 10ms and 450ms of analytics overhead on every pageview.

Self-hosting adds one more benefit: your data never leaves your server. No data sharing agreements to worry about, no Google using your traffic data to train ad models.

I use it alongside Google Search Console for blixamo.com. Plausible handles real-time traffic; GSC handles search queries and impressions. Together they replace everything I ever needed from GA4. My [Google Search Console setup guide for self-hosted Next.js](/blog/google-search-console-self-hosted-nextjs) covers the GSC side.

## Step 1 of 5 — Clone Plausible and Configure

```bash
# Create directory and clone hosting config
mkdir -p /opt/plausible && cd /opt/plausible
git clone https://github.com/plausible/hosting .
```
<Callout type="tip" title="Expected output">
Cloning into '.'...
remote: Counting objects: 47, done.
</Callout>
#
Edit the `plausible-conf.env` file — this is the only config file you need to touch:
#
```bash
# /opt/plausible/plausible-conf.env
BASE_URL=https://analytics.yourdomain.com
SECRET_KEY_BASE=GENERATE_THIS_BELOW
TOTP_VAULT_KEY=GENERATE_THIS_BELOW
```
Generate the required secret keys:
#
```bash
# Generate SECRET_KEY_BASE
openssl rand -base64 48
# Generate TOTP_VAULT_KEY (different format)
openssl rand -base64 32
```
<Callout type="tip" title="Expected output">
xK9mP2vL8nQ4rJ7wY3uI6tR1sA5bD0eF9cG2hN8kM4pQ7vL3wR6tJ1sA=
</Callout>
#
Paste the outputs into `plausible-conf.env`. Each key goes on its own line after the `=`.
#
**Why this matters:** The `SECRET_KEY_BASE` signs session tokens. If you ever change it, all existing sessions are invalidated and every user gets logged out. Generate it once and never change it.

## Step 3 of 5 — Configure Nginx Reverse Proxy
#
```bash
# /etc/nginx/sites-available/plausible
server {
    server_name analytics.yourdomain.com;
# ---
    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Host $host;
    }
# ---
    listen 80;
}
```
```bash
ln -s /etc/nginx/sites-available/plausible /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
certbot --nginx -d analytics.yourdomain.com \
  --non-interactive --agree-tos -m your@email.com
```
<Callout type="tip" title="Expected output">
Successfully received certificate.
Certificate is saved at: /etc/letsencrypt/live/analytics.yourdomain.com/fullchain.pem
</Callout>
#
If you're using Coolify, skip nginx — add Plausible as a Docker Compose resource in Coolify and it handles SSL and routing automatically. See the [Coolify complete guide](/blog/coolify-complete-guide-2026) for that workflow.

## Step 4 of 5 — Create Account and Add Your Site
#
Open `https://analytics.yourdomain.com`. You'll see the Plausible registration screen.
#
<Callout type="warning" title="Self-hosted: registration is open by default">
Anyone with your URL can register. After creating your account, disable registration by adding `DISABLE_REGISTRATION=true` to `plausible-conf.env` and running `docker compose restart plausible`.
</Callout>
#
After logging in:
1. Click **Add Website**
2. Enter your domain (e.g. `blixamo.com`)
3. Copy the tracking snippet — it looks like this:
#
```html
<!-- Add to your <head> tag -->
<script defer
  data-domain="yourdomain.com"
  src="https://analytics.yourdomain.com/js/script.js">
</script>
```
**For Next.js**, add to `app/layout.tsx`:
#
```bash
// app/layout.tsx
import Script from 'next/script'
// ---
export default function RootLayout({ children }) {
  return (
    <html>
      <head>
        <Script
          defer
          data-domain="yourdomain.com"
          src="https://analytics.yourdomain.com/js/script.js"
        />
      </head>
      <body>{children}</body>
    </html>
  )
}
```
<Callout type="tip" title="Expected output">
After deploying: visit your site, then check Plausible dashboard — you should see 1 visitor appear within 5 seconds.

**Set up data backup:**
#
Plausible's Clickhouse data is in a Docker volume. Back it up daily:
#
```bash
# Add to crontab: crontab -e
0 3 * * * docker exec plausible_events_db \
  clickhouse-client --query \
  "BACKUP DATABASE plausible_events TO Disk('backups', 'daily.zip')"
```

## Troubleshooting Plausible
#
**Problem: Dashboard shows 0 visitors despite traffic**
#
Most common cause: ad blocker or browser privacy settings blocking the script. Test in an incognito window without extensions. Also verify the script tag is in `<head>` not `<body>`.
#
```bash
# Check Plausible is receiving events
docker logs plausible --tail 50 | grep "event"
```
**Problem: Clickhouse container keeps restarting**
#
```bash
docker logs plausible_events_db --tail 30
```
Usually OOM. Add 2GB swap if not already present:
#
```bash
swapon --show
# If empty:
fallocate -l 2G /swapfile && chmod 600 /swapfile
mkswap /swapfile && swapon /swapfile
```
**Problem: Registration page still showing after disabling**
#
```bash
grep DISABLE_REGISTRATION /opt/plausible/plausible-conf.env
# If set, force restart:
docker compose down && docker compose up -d
```
**Problem: SSL cert not provisioning**
#
Cloudflare proxy must be OFF during Let's Encrypt ACME challenge. Set DNS to DNS-only (grey cloud), wait 2 minutes, then re-run certbot.

## FAQ
#
### Is Plausible really GDPR compliant without cookie consent?
#
Under current GDPR guidance, analytics that don't use persistent identifiers (cookies or device fingerprints stored across sessions) don't require consent. Plausible hashes IP + user agent + date, creating a daily identifier that resets. Most legal interpretations classify this as compliant. Consult your legal team for definitive advice specific to your situation.
#
### How accurate is Plausible vs Google Analytics?
#
Plausible typically shows 10-20% more pageviews than GA4 because it's not blocked by as many ad blockers (though some block it). GA4 also samples data on high-traffic sites — Plausible never samples. For a blog under 1M pageviews/month, Plausible's numbers are more accurate.
#
### Can I import Google Analytics historical data into Plausible?
#
Yes — Plausible has a built-in GA import tool. Go to Site Settings → Imports & Exports → Import from Google Analytics. Works for both Universal Analytics and GA4 data.
#
### How much disk space does Plausible use?
#
Clickhouse is extremely efficient for time-series data. 1 million pageviews uses roughly 50MB of disk space. On Hetzner CPX22's 80GB SSD, you can store years of analytics data without worrying about disk.
#
### Does Plausible work behind Cloudflare?
#
Yes. Set Cloudflare to DNS-only during initial SSL setup, then re-enable the proxy. Cloudflare caches Plausible's static assets but passes event tracking requests through to your origin correctly.
#
### Can I track multiple websites?
#
Yes — unlimited sites on self-hosted Plausible. I track blixamo.com and two client projects from the same dashboard. Each site has its own data silo; you can't accidentally mix data between sites.
