Next.js has become the go-to framework for React developers who want performance and SEO without sacrificing developer experience. This guide gets you up and running from zero.

## Why Next.js in 2026?

Three reasons dominate:

1. **App Router** — file-based routing with React Server Components baked in
2. **Built-in SEO** — metadata API, automatic sitemap, and OG image generation
3. **Performance by default** — image optimization, font optimization, and edge rendering

## Prerequisites

You need Node.js 18+ and a basic understanding of React. That's it.

## Setting Up Your First Project

Run the create command and follow the prompts:

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

Your app is live at `http://localhost:3000`.

## Understanding the App Router

The `app/` directory is where all your routes live. Every `page.tsx` file becomes a route:

```
app/
├── page.tsx          → /
├── about/page.tsx    → /about
└── blog/[slug]/page.tsx → /blog/anything
```

## Adding Your First Page

Create `app/about/page.tsx`:

```tsx
export default function AboutPage() {
  return <h1>About Me</h1>
}
```

Navigate to `/about` — it just works.

## Metadata and SEO

Export a `metadata` object from any page:

```tsx
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'About Me',
  description: 'Learn more about this site.',
}
```

Next.js handles the `<title>` and `<meta>` tags automatically.

## Deploying to Production

The simplest path is Vercel — push to GitHub, connect the repo, done. For a VPS (like blixamo.com), run:

```bash
npm run build
npm start
```

Use PM2 to keep it alive: `pm2 start npm --name "my-app" -- start`

**Is Next.js free?**
Yes — the framework is open source. Hosting costs depend on your provider.

**Can I use Next.js with TypeScript?**
Absolutely. It ships with TypeScript support out of the box.
