## What Is Coolify and Why Use It in 2026?

Coolify is an open-source self-hosting platform that manages Docker containers, SSL certificates, reverse proxy routing, and GitHub deployments through a web UI. You install it on any server and it handles the infrastructure plumbing that used to require manual nginx + certbot + Docker Compose configuration for every project.

As of March 2026, Coolify v4 is actively developed — new features ship every few weeks. The community on Discord has 15,000+ members. It's not a side project someone built and abandoned; it's the most actively maintained self-hosting platform available.

**Why it matters in 2026 specifically:** Managed hosting prices went up 30-40% since 2024 (Vercel, Railway, Render all raised prices). Meanwhile Hetzner's CPX22 stayed at €5.19/month. The math now strongly favours self-hosting for indie developers running multiple projects.

I compared all three major options in [Coolify vs Caprover vs Dokku](/blog/coolify-vs-caprover-2026) if you want the full breakdown. Short answer: Coolify wins for anyone who wants a UI and active development.

**What Coolify handles for you:**
- Automatic Let's Encrypt SSL for every domain
- Traefik reverse proxy — route multiple domains to different apps on one server
- GitHub webhook auto-deploy on every `git push`
- One-click databases: PostgreSQL, MySQL, Redis, MongoDB, MariaDB
- S3-compatible backups for databases
- Container isolation between apps

## Step 2 of 6 — Harden the VPS Before Installing Coolify
#
Before Coolify touches your server, lock it down. When I set up my first Hetzner VPS in 2025, SSH brute-force attempts started within 3 hours of creation. These two steps stop 99% of attacks.
#
```bash
# Update packages first
apt update && apt upgrade -y
# ---
# Install fail2ban — blocks IPs after repeated failed SSH attempts
apt install fail2ban -y
systemctl enable fail2ban
systemctl start fail2ban
# ---
# Add 2GB swap — critical for Next.js builds on 4GB VPS
# Without this you will hit OOM during builds (I hit 13 in one week)
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```
<Callout type="tip" title="Expected output">
Setting up swapspace version 1, size = 2 GiB (2147479552 bytes)
</Callout>
#
The swap step is not optional. Next.js builds on a 4GB VPS peak at ~3.2GB RAM. Without swap, PM2 or Docker containers OOM-kill mid-build. I learned this the hard way — 13 OOM restarts in one week before I added the swap. Full security hardening details are in my [VPS security guide for Ubuntu 2026](/blog/vps-security-harden-ubuntu-2026).

## Step 3 of 6 — Install Coolify
#
One command installs Docker, configures Traefik, sets up the Coolify database, and starts all services:
#
```bash
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash
```
<Callout type="tip" title="Expected output">
Coolify installed successfully!
Open http://YOUR_SERVER_IP:8000 to continue.
</Callout>
#
The installer takes 3-5 minutes. It installs Docker Engine, pulls the Coolify image stack (coolify, traefik, postgres, redis), and starts everything via Docker Compose.
#
```bash
# Verify Coolify is running after install
docker ps --format 'table .Names .Status'
```
<Callout type="tip" title="Expected output">
coolify          Up 2 minutes
coolify-db       Up 2 minutes
coolify-redis    Up 2 minutes
coolify-proxy    Up 2 minutes

Open `http://YOUR_SERVER_IP:8000` in your browser. You'll see the Coolify registration screen. Create your admin account.

<Callout type="warning" title="Port 8000 must be open">

## Step 4 of 6 — Connect Your Domain and GitHub
#
**Add your server to Coolify:**
#
In the Coolify dashboard → **Servers** → your server is already added as localhost. Click it → **Verify** to confirm Docker connectivity.
#
**Add your GitHub:**
#
Go to **Sources** → **Add** → **GitHub App**. Follow the OAuth flow. Coolify creates a GitHub App in your account with webhook permissions — this enables auto-deploy on every `git push`.
#
**Point your domain to the server:**
#
In your DNS provider (Cloudflare recommended), add:
#
```text
A    @         YOUR_SERVER_IP    Proxied: OFF initially
A    *         YOUR_SERVER_IP    Proxied: OFF initially
```
<Callout type="warning" title="Cloudflare: turn off proxy during SSL provisioning">
Set DNS to DNS-only (grey cloud) when provisioning Let's Encrypt certs. Cloudflare's orange cloud mode interferes with the ACME challenge. After cert is issued, switch back to proxied.
</Callout>

## Step 5 of 6 — Deploy Your First Next.js App
#
**Create a new project:**
#
Coolify → **Projects** → **Add** → **New Project** → name it.
#
**Add a resource (your app):**
#
→ **Add Resource** → **Public Repository** (or Private if you connected GitHub) → paste your repo URL → select branch (`main`) → click **Continue**.
#
Coolify detects Next.js via Nixpacks automatically. It sets the build command (`npm run build`) and start command (`node server.js`) without you touching anything.
#
**Set your domain:**
#
In the app settings → **Domains** → add `yourdomain.com` → click **Save**. Coolify provisions Let's Encrypt SSL automatically within 2 minutes.
#
**Deploy:**
#
Click **Deploy**. Watch the build logs in real time. For a standard Next.js 15 app, first build takes 3-4 minutes on CPX22.
#
```bash
# You can also watch container logs from SSH if needed
docker logs coolify-proxy --tail 50 -f
```
<Callout type="tip" title="Expected output">
Build completed successfully.
Container started: myapp_production
Domain: https://yourdomain.com — SSL active
</Callout>
#
**Enable auto-deploy:**
#
In app settings → **General** → toggle **Auto Deploy** on. Every `git push main` now triggers a build and deploy automatically via GitHub webhook.
#
This is the workflow I use for all client projects. Push to main, Coolify builds, zero SSH required.

## Step 6 of 6 — Add a PostgreSQL Database
#
Coolify → **Project** → **Add Resource** → **Database** → **PostgreSQL** → select version (16 recommended) → click **Deploy**.
#
Coolify creates an isolated PostgreSQL container with persistent storage mounted at `/var/lib/docker/volumes/`.
#
```bash
# Connection string format Coolify gives you:
# postgresql://postgres:PASSWORD@HOST:PORT/postgres
```
**Connect it to your app:**
#
In your app's **Environment Variables** section, add:
#
```bash
DATABASE_URL=postgresql://postgres:PASSWORD@coolify-db-container:5432/postgres
```
<Callout type="tip" title="Expected output">
Environment variables saved. Redeploy to apply.
</Callout>
#
**Enable S3 backups (optional but recommended):**
#
Database settings → **Backups** → add your S3-compatible bucket credentials. Coolify runs `pg_dump` on schedule and uploads the compressed dump. Backfill recovery from a Cloudflare R2 bucket costs essentially zero.

## Troubleshooting Common Coolify Problems
#
<Callout type="warning" title="Real issues I've hit in 6 months of production use">
These are actual problems from my setup — not theoretical edge cases.
</Callout>
#
**Problem: Build fails with "out of memory" during Next.js build**
#
```bash
# Check available memory during build
free -h
# If available < 500MB, the build will OOM
# ---
# Fix: verify swap is active
swapon --show
# If empty, re-add swap:
swapon /swapfile
```
**Problem: SSL certificate fails to provision**
#
Most likely cause: Cloudflare proxy is ON during cert provisioning. Set DNS to DNS-only, wait 5 minutes for propagation, then click **Renew Certificate** in Coolify.
#
**Problem: Auto-deploy webhook not triggering**
#
```bash
# Check Coolify webhook logs
docker logs coolify --tail 100 | grep webhook
```
Usually caused by GitHub App permissions expiring. Go to **Sources** → your GitHub App → **Reconnect**.
#
**Problem: Traefik routing shows 502 after deploying**
#
```bash
# Check if your container actually started
docker ps | grep your-app-name
# If not running:
docker logs your-app-container-name --tail 50
```
99% of the time this is a missing environment variable causing the app to crash on startup.
#
**Problem: Coolify dashboard shows server as "unreachable"**
#
```bash
# Restart the Coolify stack
cd /data/coolify/source
docker compose restart
```
This happens after VPS reboots if Docker doesn't come back up cleanly. Adding `--restart=always` to the Coolify compose handles this in v4+.

## Decision Framework — When to Use Coolify vs Manual Setup
#
**Use Coolify for:** Side projects, client apps, n8n, Ghost, any app where you want auto-deploy from GitHub without SSH access.
#
**Use manual PM2+nginx for:** Apps needing custom nginx config (ISR + Cloudflare CDN, specific Cache-Control headers, complex rate limiting rules).
#
**Don't use Coolify for:** Apps that need to control ports 80/443 themselves (conflicts with Traefik), or apps with complex multi-container architectures better handled by raw Docker Compose.

## FAQ
#
### How much RAM does Coolify itself use?
#
Coolify's own stack (coolify, traefik, postgres, redis containers) uses ~400MB at idle on a fresh install. On a 4GB CPX22, that leaves ~3.3GB for your apps. I run 5 apps and databases in the remaining space with 2.7GB headroom.
#
### Can I use Coolify with Cloudflare?
#
Yes, but disable Cloudflare's proxy (orange cloud) during SSL certificate provisioning. After the cert is issued, re-enable Cloudflare proxy. Coolify handles the origin certificate; Cloudflare handles CDN and DDoS protection at the edge.
#
### Does Coolify work on a 2GB VPS?
#
Barely. Coolify's own stack uses ~400MB, leaving ~1.4GB for your apps. Next.js builds will OOM without swap. Use a 4GB VPS minimum. Hetzner CPX22 at €5.19/month is the right size.
#
### How do I update Coolify?
#
From the Coolify dashboard → **Settings** → **Updates** → click **Update**. Coolify pulls the latest image and restarts. Takes ~2 minutes. Your apps keep running during the update.
#
### Is my data safe if the VPS reboots?
#
Yes. Coolify mounts volumes at `/var/lib/docker/volumes/` which persists across container restarts and reboots. Database data survives VPS reboots. Configure S3 backups for off-server redundancy.
#
### Can I self-host Coolify on a non-Hetzner server?
#
Yes. Coolify works on any Ubuntu/Debian server. Hetzner is just the best price-to-performance ratio I've found — €5.19/month for 4GB RAM vs Digital Ocean's $19/month for 2GB.
#
### What happens when Coolify itself goes down?
#
Your apps keep running — they're Docker containers managed by Docker's own restart policy. Only new deployments stop working. Restart Coolify with `docker compose restart` in `/data/coolify/source`.
