One VPS. Five apps. One IP address. Nginx routes traffic to the right app based on domain name — with SSL, Cloudflare CDN, and automatic certificate renewal.

This is the exact setup I run on my Hetzner CPX22 (Ubuntu 24.04, €5.19/month). blixamo.com, n8n, Plausible Analytics, and two client projects all share one server. Each has its own domain, its own SSL certificate, and its own nginx server block.

No Coolify, no Traefik, no Docker Compose proxy — just nginx configured the right way. The config in this guide is production-tested and handles blixamo.com's traffic with a 306ms average response time globally.

## Step 2 of 5 — Basic Server Block (One App)
#
Create a server block for your first app. This example is for a Next.js app running on port 3000:
#
```bash
# /etc/nginx/sites-available/blixamo
server {
    server_name blixamo.com www.blixamo.com;
# ---
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
# ---
    location /_next/static/ {
        proxy_pass http://localhost:3000;
        add_header Cache-Control "public, max-age=31536000, immutable";
    }
# ---
    listen 80;
}
```
```bash
# Enable the site
ln -s /etc/nginx/sites-available/blixamo /etc/nginx/sites-enabled/
# ---
# Test config syntax
nginx -t
# ---
# Reload nginx
systemctl reload nginx
```

## Step 3 of 5 — SSL with Let's Encrypt
#
Point your domain's DNS A record to your VPS IP first. With Cloudflare, set DNS to DNS-only (grey cloud) during SSL provisioning:

## Step 4 of 5 — Multiple Apps on One Server
#
Each additional app needs its own server block. Here's the full multi-app setup:
#
**n8n (Docker on port 5678):**
#
```bash
# /etc/nginx/sites-available/n8n
server {
    server_name n8n.yourdomain.com;
# ---
    location / {
        proxy_pass http://localhost:5678;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        # Required for n8n websocket connections
        proxy_read_timeout 86400;
    }
    listen 80;
}
```
**Plausible Analytics (Docker on port 8000):**
#
```bash
# /etc/nginx/sites-available/plausible
server {
    server_name analytics.yourdomain.com;
# ---
    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Host $host;
    }
    listen 80;
}
```
**Enable all sites and get SSL for each:**
#
```bash
ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/
ln -s /etc/nginx/sites-available/plausible /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
# ---
certbot --nginx \
  -d n8n.yourdomain.com \
  -d analytics.yourdomain.com \
  --non-interactive --agree-tos -m your@email.com
```

## Step 5 of 5 — ISR + Cloudflare CDN Cache Headers
#
This is the config that took blixamo.com from 2 seconds to 306ms load time. Standard nginx proxy configs don't set the right cache headers for Cloudflare CDN to cache ISR pages.
#
After getting SSL, update your Next.js server block with these cache headers:
#
```bash
# /etc/nginx/sites-available/blixamo (full production config)
server {
    server_name blixamo.com www.blixamo.com;
# ---
    # Main app — ISR cache headers for Cloudflare
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # Override Next.js default cache headers for Cloudflare ISR
        proxy_hide_header Cache-Control;
        add_header Cache-Control "public, max-age=0, s-maxage=3600, stale-while-revalidate=86400";
    }
# ---
    # Next.js static assets — cache forever (content-addressed filenames)
    location /_next/static/ {
        proxy_pass http://localhost:3000;
        add_header Cache-Control "public, max-age=31536000, immutable";
    }
# ---
    # Images served from /public — cache 1 week
    location /images/ {
        proxy_pass http://localhost:3000;
        add_header Cache-Control "public, max-age=604800";
    }
# ---
    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/blixamo.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/blixamo.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;
}
# ---
# HTTP to HTTPS redirect
server {
    listen 80;
    server_name blixamo.com www.blixamo.com;
    return 301 https://$host$request_uri;
}
```
```bash
nginx -t && systemctl reload nginx
```
<Callout type="tip" title="Expected output">

nginx: configuration file test is successful
</Callout>

## Performance Tuning
#
Add these to `/etc/nginx/nginx.conf` in the `http` block for production performance:
#
```bash
# /etc/nginx/nginx.conf — http block additions
http {
    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/javascript
               application/javascript application/json
               application/xml image/svg+xml;
# ---
    # Brotli (install nginx-module-brotli for this)
    # brotli on;
# ---
    # Keep connections alive — reduces TCP handshake overhead
    keepalive_timeout 65;
    keepalive_requests 100;
# ---
    # Buffer sizes — prevent slow client attacks
    client_body_buffer_size 10K;
    client_header_buffer_size 1k;
    client_max_body_size 8m;
    large_client_header_buffers 2 1k;
# ---
    # Timeouts
    client_body_timeout 12;
    client_header_timeout 12;
    send_timeout 10;
}
```

## Troubleshooting Nginx
#
**Problem: 502 Bad Gateway**
#
Your app isn't running on the specified port:
#
```bash
# Check what's actually running on the port
netstat -tlnp | grep :3000
# If empty — your app crashed. Check app logs:
pm2 logs blixamo --lines 50
```
**Problem: SSL certificate fails to provision**
#
Cloudflare proxy must be OFF (grey cloud) during Let's Encrypt provisioning. The ACME HTTP challenge can't complete through Cloudflare's proxy.
#
```bash
# Verify port 80 is reachable from outside
curl -I http://yourdomain.com
# Should return 200 or redirect — not timeout
```
**Problem: `nginx -t` fails after editing config**
#
```bash
# Find the specific error
nginx -t 2>&1
# Common mistakes: missing semicolon, unclosed block, wrong path
```
**Problem: Cloudflare showing CF-Cache-Status: BYPASS**
#
A cookie in the response is bypassing cache. Check for any `Set-Cookie` headers from your app. Cloudflare bypasses cache when cookies are present by default:
#
```bash
curl -I https://yourdomain.com | grep -i "set-cookie\|cf-cache"
```
Fix: In Cloudflare → Caching → Cache Rules → create a rule to cache even when cookies present for your blog paths.
#
**Problem: WebSocket connections failing (n8n)**
#
Missing WebSocket upgrade headers. Ensure your n8n server block has:
#
```bash
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection 'upgrade';
proxy_read_timeout 86400;
```

## FAQ
#
### Can nginx handle 10,000 requests per day on a 4GB VPS?
#
Easily. Nginx itself uses ~5MB RAM and handles tens of thousands of concurrent connections. At 10,000 requests/day (average ~7 requests/minute), nginx overhead is negligible. The bottleneck is your app's processing time, not nginx.
#
### Should I use Nginx or Coolify's Traefik for reverse proxy?
#
Use Coolify/Traefik if you want a web UI and don't need custom cache headers. Use manual nginx if you need precise control over Cache-Control headers (required for ISR + Cloudflare CDN). I use manual nginx for blixamo.com specifically because ISR + Cloudflare CDN requires the `s-maxage` and `stale-while-revalidate` headers that Traefik doesn't set correctly by default.
#
### How do I add a new app to an existing nginx setup?
#
Create a new file in `/etc/nginx/sites-available/`, symlink to `sites-enabled/`, run `nginx -t`, reload nginx, then run certbot for the new domain. Takes 5 minutes per app.
#
### Does nginx work with Cloudflare's orange cloud proxy?
#
Yes — and it's recommended. Set your A record to Proxied (orange cloud) after SSL is provisioned. Cloudflare terminates HTTPS at the edge and proxies to your origin over HTTP or HTTPS. Your nginx still needs a valid SSL certificate even with Cloudflare in front.
#
### What is the difference between `proxy_pass` and a redirect?
#
`proxy_pass` forwards the request to another server — the client sees your domain, not the backend's port. A redirect (`return 301`) sends the client to a different URL — the client's browser makes a new request to the new URL and sees it. For reverse proxying apps, always use `proxy_pass`.
