## n8n — Best for Indie Developers Who Can Self-Host

Self-hosted n8n on Hetzner is the clear winner for indie developers worldwide comfortable with the command line. The cost equation is simple: $0/month for unlimited workflows versus $82/month on Zapier for the same automations.

I run n8n in Docker on the same CPX22 VPS that hosts blixamo.com, a WhatsApp AI assistant, and three other projects. RAM usage for n8n in queue mode is ~180MB. It runs alongside Next.js (PM2), Redis, and a webhook handler — total VPS RAM used: 715MB of 4GB.

The workflow I use most: new MDX file saved → inotifywait detects it → n8n fires → posts a tweet + sends a GSC indexing request + logs to Supabase. This runs hundreds of times per month at zero marginal cost.

- UI is less polished than Make or Zapier
- Debugging failed workflows means reading JSON logs
</Callout>

**Who should use n8n:** Any indie developer comfortable with a VPS. If you already run Hetzner + Docker, n8n adds ~15 minutes of setup and costs nothing extra, and the [n8n complete guide](/blog/n8n-complete-guide-2026) is the better deep dive if you want the platform itself explained before you compare it against Make or Zapier.

**Who should use Make:** Non-technical founders, content creators, and small teams who need reliable automation without VPS management overhead. If your time is worth more than $9/month, Make is worth it.

Zapier is the most polished automation platform with the largest integration library at 6,000+ apps. But at $28–7,400/month for any meaningful usage, it's priced for funded startups and enterprise teams — not indie developers worldwide bootstrapping on tight margins.

The free tier limit of 100 tasks/month is essentially unusable. A single automation running on every new email would burn through that in days.

<Callout type="danger" title="Zapier Free Tier Reality">

Zapier's only genuine advantage over Make is integration breadth. If you need a very niche SaaS tool that only has a Zapier integration, you may have no choice. For everything mainstream — Google Sheets, Notion, Slack, Gmail, webhooks — Make covers it at half the price.

**Who should use Zapier:** Teams with $50+/month budget who need a specific integration only Zapier has, or enterprises where Zapier's polish justifies the cost.

## Troubleshooting Common Issues
#
#
![Zapier dashboard showing $80/month plan needed for 5 automations vs n8n free self-hosted](/images/posts/n8n-vs-make-vs-zapier-indie-dev/04-zapier-dashboard.png)

#
### n8n Docker Container Keeps Restarting
#
Most likely cause (65%): memory limit too low. Docker has no memory limit by default — set `--memory=512m` in your compose file. That prevents n8n from consuming available RAM during heavy workflow execution.
#
Second most likely (25%): SQLite database corruption from an unclean shutdown. Fix it with:
#
```bash
docker exec n8n n8n db:revert
docker restart n8n
```
<Callout type="tip" title="Expected output">
#
(command runs silently — no output means success)
#
</Callout>
### Make Operations Running Out Too Fast
#
Check your workflow structure. Each module (step) in a Make scenario costs 1 operation — a 5-step workflow costs 5 operations per run, not 1. Consolidate steps using the "Router" module and use filters to skip unnecessary branches.
#
### Zapier Webhook Not Triggering
#
Zapier's free webhooks only work with "instant" triggers. If you're on a paid plan and webhooks still fail, check two things:
#
- Payload is under 15MB
- Response time is under 30 seconds (Zapier times out at 30s)

## Setting Up n8n in 30 Minutes
#
If you're convinced on n8n, here's the fastest path to running it on Hetzner:
#
```bash
# Install Docker (if not already present)
curl -fsSL https://get.docker.com | sh
#
# Create data directory
mkdir -p /root/n8n-data
#
# Run n8n
docker run -d \
  --name n8n \
  --restart unless-stopped \
  -p 5678:5678 \
  -e N8N_SECURE_COOKIE=false \
  -v /root/n8n-data:/home/node/.n8n \
  n8nio/n8n
```
<Callout type="tip" title="Expected output">
`docker ps | grep n8n` should show the container with status `Up X minutes`. Open `http://YOUR_VPS_IP:5678` to reach the n8n setup wizard. If it doesn't load, check `docker logs n8n`.
</Callout>

## Switching From Zapier or Make to n8n
#
Migration is manual — there's no automated migration tool between these platforms. The practical approach:
#
1. Export your existing workflow logic as documentation (screenshot or describe each trigger → action chain)
2. Set up n8n (local or Hetzner) and rebuild the top 3 most valuable workflows first
3. Run both systems in parallel for 2 weeks — let n8n handle the rebuilt workflows, keep the old platform running everything else
4. Once the n8n workflows are stable, cancel the old subscription
#
Rebuilding a typical Zapier workflow in n8n takes 20–45 minutes for simple linear flows, 1–2 hours for complex branching logic. The learning curve is front-loaded — after building 5 workflows in n8n, the 6th takes half as long.
#
The one thing that doesn't migrate: institutional knowledge about why certain edge cases are handled a particular way. Document your Zapier/Make workflows before rebuilding them in n8n or you'll rebuild the same bugs.
