## TL;DR

- Docker Compose with n8n, FastAPI, PostgreSQL, Redis.
- Use Hetzner CPX22 (4GB) with UFW, swap, and fail2ban.
- Expose n8n/ FastAPI through Nginx reverse proxy + SSL via Certbot.
- Automate health checks using [Best VPS Monitoring Tools in 2026](/blog/best-vps-monitoring-tools-2026).
- Pair with [Ubuntu VPS Hardening Checklist](/blog/ubuntu-vps-hardening-checklist) for security.

## Architecture overview

1. **Hetzner CPX22** as the host, snapshot schedule in place.
2. **Docker Compose** stack (n8n, FastAPI, PostgreSQL, Redis, Traefik or Nginx).
3. **Reverse proxy** handles SSL, domain, and rate limiting.
4. **Monitoring + automation** send alerts to Telegram/Slack.
5. **Automated backups** to Hetzner snapshots or object storage.

## Compose snippet

```yaml
version: "3.9"

services:
  postgres:
    image: postgres:15
    volumes:
      - ./data/postgres:/var/lib/postgresql/data
    environment:
      POSTGRES_PASSWORD: secure

  redis:
    image: redis:7

  fastapi:
    build: ./fastapi
    depends_on:
      - postgres
      - redis
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: postgres://postgres:secure@postgres:5432/fastapi

  n8n:
    image: n8nio/n8n:latest
    ports:
      - "5678:5678"
    environment:
      DB_TYPE: postgres
      DB_POSTGRESDB_HOST: postgres
      DB_POSTGRESDB_PASSWORD: secure
      EXECUTIONS_PROCESS: main

  nginx:
    build: ./nginx
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - fastapi
      - n8n
```

## Post-deploy steps

- Run Certbot for `n8n.example.com` and `api.example.com`.
- Add UFW rules per [Ubuntu VPS Hardening Checklist](/blog/ubuntu-vps-hardening-checklist) (ports 80,443,5678,8000).
- Enable Hetzner monitoring + UptimeRobot from [Best VPS Monitoring Tools in 2026](/blog/best-vps-monitoring-tools-2026).
- Automate health checks via `n8n` (see [Self-Healing VPS Monitor 2026](/blog/self-healing-vps-monitor-nodejs)).
