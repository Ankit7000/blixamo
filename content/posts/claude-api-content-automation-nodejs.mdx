## Step 5 of 8 â€” AI Fixes: What Claude Haiku Generates
The six AI fixes each make one API call to Claude Haiku. I use Haiku not Sonnet because these are mechanical rewrites following explicit format rules â€” Haiku handles them reliably at 1/5th the cost.
**Hook rewrite** â€” checks if the first paragraph starts with "I " and is under 45 words. If not, rewrites it:

### How much does the daily fixer run cost in Claude API tokens?
A full run across 15 articles costs roughly $0.04â€“0.08 (Rs 3â€“7). Claude Haiku costs $0.80/million input tokens and $4/million output tokens. Each AI fix call uses ~300â€“600 input tokens and generates 100â€“300 output tokens. The initial backlog run across 15 articles cost under $0.50 (Rs 42) total.
### Can this run on any blog or is it blixamo-specific?
The 30 ARTICLE_RULES and the context builder are blixamo-specific. The architecture â€” deterministic fixes first, AI fixes second, flag-only for human judgment third â€” is generic. To adapt it: replace `REAL_DATA` in `blixamo-context-builder.js` with your own production numbers and failures. Replace `LIVE_ARTICLES` with your article map. The fix functions themselves are reusable as-is.
### Does the fixer modify articles that are already correct?
No. Every fix function has a guard that returns `null` if the check passes. `fixTLDR()` returns null if `raw.includes('TL;DR')`. `fixHook()` returns null if the hook already starts with "I " and is under 45 words. The fixer is idempotent â€” running it twice on an already-clean article applies zero changes.
### What happens if Claude returns bad output?
Every AI fix validates the response before applying it. `fixHook()` rejects any output that doesn't start with "I " or "My ". `fixTLDR()` rejects output that doesn't contain "TL;DR". The worst case is the fix is skipped and logged as a warning â€” the article is never corrupted.
### How is this different from tools like Clearscope or Surfer SEO?
Those tools analyse content against competitor pages and suggest keywords. This tool enforces a specific internal standard â€” ARTICLE_RULES.md â€” independent of competitor analysis. It fixes structural and voice issues, not keyword density. The focus is on AI Overview citation probability (SCU blocks, FAQ structure) rather than traditional keyword SEO.
### Does it work with `next-mdx-remote` frontmatter?
Yes. The fixer reads and writes raw MDX files directly using `fs.readFileSync` and `fs.writeFileSync`. Changes are picked up by the `blixamo-watcher` service, which fires ISR revalidation within 1 second â€” the fixed article is live on the site within seconds of the fixer writing the file.
