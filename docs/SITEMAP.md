## Sitemap Generation Logic (app/sitemap.ts)

Includes:
1. / (homepage)
2. /about
3. /contact
4. /privacy-policy
5. /terms
6. /disclaimer
7. /category/[slug] for every category that has at least 1 post
8. /tag/[tag] for each of 13 KEEP_TAGS only
9. /blog/[slug] for all 44 posts

Does NOT include:
- /blog (index page)
- /search
- /api/* routes
- /author/* pages
- /ads.txt
- Tags not in KEEP_TAGS

lastModified behavior:
- homepage uses the freshest post `updatedAt` or `date` 
- category pages use the freshest post inside that category
- tag pages use the freshest post carrying that tag
- blog post pages use `updatedAt` if set, otherwise `date` 
- legal/static pages use a fixed document-level date until those pages are edited

## URL Rules

- Post slug = MDX filename without extension. Must be lowercase, hyphen-separated.
- Category slug = raw frontmatter value. Must exactly match one of the 9 registered slugs.
- Tags = raw values from tags[] array. Lowercase, hyphen-separated. Reuse consistently.
- Author slug = author.toLowerCase() from frontmatter.
- Never rename a slug after Google has indexed it.
